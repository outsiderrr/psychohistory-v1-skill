# Psychohistory · 宏观事件推演引擎 🌳

[English](./README.md) · **中文**

> 不是水晶球，是结构化思维工具。

输入一个正在发生的复杂事件，输出多Agent博弈分析和可能性演化树。

帮你穷举关键决策者、硬约束、因果链和可能的走向——避免遗漏，而非预测未来。

## 看效果

**输入：** "帮我推演美伊谈判破裂后的局势走向"

**输出：**

```

├── ① 停火到期后战事恢复 ⭐ Rank 1
│   ├── [GT] 三大症结均触及核心利益，博弈空间极窄
│   ├── [PSY] 特朗普的「把一切个人化」已激活，退让阈值极高
│   └── [ORG] 以色列持续施压，充当谈判破坏者
│
├── ② 未签署，但停火事实延续 
│   ├── [GT] 双方都有避免全面升级的动机
│   └── [ORG] 大规模军事行动重启需要后勤准备时间
│
└── ③ 签署停火延长协议
    ├── [GT] 三大症结（海峡、浓缩铀、资产解冻）均未解决
    └── [PSY] 万斯称已提出「最终提议」，伊朗无进一步谈判计划
```

每条分支标注推理来源：`[GT]` 博弈论 · `[PSY]` 心理模型 · `[ORG]` 组织行为学

## 它做什么

1. **识别关键Agent** — 谁在做决策？谁影响决策者？他的认知框架是什么？
2. **映射硬约束** — 什么是所有人都无法改变的物理和制度边界？
3. **三引擎分析** — 博弈论、心理模型、组织行为学各自怎么看？
4. **生成可能性树** — 按可能性从高到低排列互斥选项，每条附带理由
5. **持续交互** — 注入新信息、添加新选项、修改Agent画像

## 它不做什么

- ❌ 不预测未来
- ❌ 不给出精确概率数字
- ❌ 不假装无所不知

## 适用场景

- **地缘政治** — 战争走向、外交博弈、制裁效果推演
- **宏观经济** — 央行政策、贸易战、能源市场推演
- **商业竞争** — CEO决策分析、行业格局变化、并购博弈
- **政策分析** — 立法进程、监管变化、选举推演

## 核心概念

### Agent（智能体）

所有有独立决策权的行动者。分为：
- **实体Agent** — 个人或机构负责人（特朗普、内塔尼亚胡）
- **群体Agent** — 按需引入的群体力量（MAGA基本盘、华尔街）

### 忠诚度关系

同一阵营内的Agent通过忠诚度连接。忠诚度是动态变量：
- **忠诚模式** — A可以代表整个阵营，但B的倾向是隐性约束
- **独立模式** — 忠诚度跌破阈值，B成为独立Agent

### 三引擎

| 标签 | 引擎 | 看什么 |
|------|------|--------|
| `[GT]` | 博弈论 | 收益矩阵、纳什均衡、可信威胁 |
| `[PSY]` | 心理模型 | 认知框架、偏差、情绪动量 |
| `[ORG]` | 组织行为学 | 惯性、内部摩擦、路径依赖 |

### 可能性树

每个节点是一个 Polymarket 式精确命题（二元结果、公认事实、明确时间窗口、精确描述）。分支按可能性排序，不量化概率。

## 与女娲.skill协作

Psychohistory 的Agent认知框架可以直接引用 [女娲.skill](https://github.com/alchaincyf/nuwa-skill) 蒸馏的人物Skill。

女娲负责蒸馏"这个人怎么想"，Psychohistory负责让蒸馏出来的人物在约束条件下博弈。

已有可直接使用的人物：特朗普、马斯克、张一鸣、芒格等 13 位。

## 人物卡系统

Psychohistory 使用标准化的 JSON 人物卡来建模关键决策者的认知框架。

**三层加载机制：**
1. **优先读取官方库** — 检查 `characters/` 目录下是否有预置卡片
2. **CLI环境自动生成** — 如果缺失且在命令行环境中，自动调用女娲.skill蒸馏
3. **对话环境手动提供** — 如果缺失且在网页对话中，引导用户粘贴卡片或用自然语言描述

所有人物卡，无论来源，都必须通过统一的Schema校验。

## 用户交互

推演完成后支持四种交互：

**注入已发生的事实：** "刚出了一条新闻——伊朗试射高超音速导弹" → 永久更新主树的排序和推导过程

**假设推演：** "如果伊朗对以色列发动大规模导弹袭击会怎样？" → 生成临时假设分支树，不改变主树

**添加新选项：** "你漏掉了一种可能——伊朗可能对海峡收通行费" → 系统评估新选项的排序位置

**修改Agent画像：** "我觉得特朗普还有一个特点——他把控制资源视为胜利" → 系统检查一致性后纳入分析

## 设计理念

本项目受阿西莫夫「心理史学」启发，但做的不是科幻式预言，而是工程化的结构性思维辅助。

核心信念：**不是帮你算出答案，而是帮你问对问题。**

## Roadmap

### 已实现（当前阶段）

- [x] 协议层 Schema —— agents、relationships、hard constraints、events、可能性树
- [x] 角色卡系统 —— 标准化 JSON 卡片 + Schema 校验
- [x] 角色卡生成工具包（`character-toolkit/`）—— 个人 / 组织 / 群体 / 关系的生成提示词
- [x] 正向发散树 + 按可能性序数排序
- [x] 三引擎分析（[GT] / [PSY] / [ORG]），历史先例优先
- [x] 事实注入 + 假设推演 两种交互模式
- [x] 事件解释层（同一原始事件经不同 agent 的认知框架过滤出不同解读）
- [x] 分支博弈闭环校验（对 rank 1-2 的分支做 Nash 可信性检查）
- [x] 分支时间尺度（resolution_horizon—— 不同时间尺度的分支不互相排名）
- [x] 角色卡的 `alternative_frameworks` 结构钩子 —— 为理论验证模式预埋接口
- [x] 群体 agent 的 `observable_state` 字段 —— 为群体一等事件节点预埋接口

### 规划中 —— 完整愿景见 [engine/README_CN.md](../engine/README_CN.md)

以下功能需要程序化执行，超出 markdown 提示词本身的能力：

- [ ] **反向推理模式** —— 给定一个目标结果，反向追溯哪些前置事件链会显著提升其发生概率。这是一个独立的推理子系统，不是正向树的反着跑
- [ ] **理论验证模式** —— 用户提供理论，系统将其表达为某个 agent 的认知框架扰动（通过 `alternative_frameworks` 字段），用扰动版和原版分别跑同一场景，对照历史事实的解释力和未来预测差异
- [ ] **群体 agent 升级为一等因果节点** —— 当 `observable_state` 的某个指标跨越战略相关阈值时，自动提升为树上的一等 Event 节点，让市场恐慌、情绪相变、社会级联有独立的结构化位置，而不是隐藏在实体 agent 的决策函数里
- [ ] 独立 Web 应用（可视化可能性树 + 交互式场景规划）
- [ ] 与女娲.skill 的深度集成
- [ ] **官方角色卡库（Official Agent Card Library）** —— 社区维护的主要地缘角色高质量角色卡集合（国家政府、领导人、组织、群体）。每张卡有 `data_cutoff` 时间戳并定期更新。用户通过 PR 贡献新卡或更新（附带证据链）。卡库让所有 Psychohistory skill 无需从零生成即可快速启动分析
- [ ] **增量更新协议（Incremental Card Update Protocol）** —— 对已有角色卡做增量刷新的工作流：Research Hand-off 只查 delta 期间的新事件，追加到 references.md §2，重新跑模式提炼，重编 JSON。~15-30 分钟 vs 全量生成的 ~1 小时
